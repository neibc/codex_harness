# Contributing to codex_harness

Thank you for considering a contribution. This project ports the [`revfactory/harness`](https://github.com/revfactory/harness) Claude Code plugin to OpenAI Codex CLI. Contributions, bug reports, and discussion are welcome.

The project is **dual-runtime**: the same repository ships a Codex plugin (root `skills/`, `agents/`, `mcp-team-server/`, `.codex-plugin/`, `.agents/`, `.mcp.json`, `hooks/`, `AGENTS.md`) and the Claude Code development harness that builds it (under `.claude/`). Please read [`README.md`](README.md) "Overview / 개요" before opening a substantive PR.

---

## Quick start

```bash
git clone https://github.com/<your-org>/codex_harness.git
cd codex_harness

# Build the MCP team-emulation server
cd mcp-team-server && npm install && npm run build && cd ..

# Run the smoke test
./tests/smoke.sh
```

The smoke test must pass (`PASS=N FAIL=0 OK`) before any PR is merged.

## How to contribute

### Reporting issues

- Use the issue tracker for bugs, missing features, or porting gaps.
- For Codex CLI version mismatches, include the output of `codex --version`. The current empirical baseline is documented in `_workspace/01_codex_primitives.md` (regenerated by the dev harness, not committed).
- For unexpected MCP team-server behavior, include a minimal reproducer that hits one of the eight tools (see [`mcp-team-server/README.md`](mcp-team-server/README.md)).

### Proposing changes

Three categories of change have different review paths:

1. **Documentation only** (typo, clarification, translation polish in `README.md` / `LIMITATIONS.md` / `AGENTS.md`): just open a PR.
2. **Codex-side artifacts** (`skills/harness/**`, `agents/**`, `mcp-team-server/**`, manifests, `hooks/**`, `tests/**`): open a PR. Please update or extend `tests/smoke.sh` so it covers your change.
3. **Mapping or architectural change** (e.g., adding a new MCP tool, supporting a new Codex hook event, changing the manifest schema): please open an issue first to discuss. These changes also require a regression of the dev-side mapping (`.claude/skills/codex-harness-orchestrator`); see [`.claude/skills/codex-harness-orchestrator/references/regression-protocol.md`](.claude/skills/codex-harness-orchestrator/references/regression-protocol.md).

### Codex CLI version updates

When a new Codex CLI version ships:

1. Re-run the dev harness in Claude Code: `> /codex-harness-orchestrator codex 하네스 빌드해줘`.
2. Compare the new `_workspace/01_codex_primitives.md` against the previous one (`_workspace/_archive/`).
3. Apply only the diffs into the Codex-side files.
4. Update `LIMITATIONS.md` if any item moved from "Unknown" to confirmed (or the reverse).
5. Bump the version in `.codex-plugin/plugin.json` and `.agents/plugins/marketplace.json`.

## Coding conventions

### TypeScript (`mcp-team-server/`)

- Target: ES2022, module: NodeNext, strict mode on.
- Keep import paths with `.js` extensions (NodeNext ESM).
- All eight tool handlers in `src/tools.ts` must validate input shape before touching `storage.ts`.
- SQLite mutations are append-only where possible (`messages` table is never UPDATE/DELETE; `tasks` mutations are recorded in `task_history`). Do not weaken this contract.

### Markdown (`skills/`, `agents/`, top-level docs)

- Frontmatter (YAML) on `SKILL.md` and agent files is required. `name` and `description` are required keys.
- Keep `SKILL.md` body under ~500 lines; move detail into `references/`.
- Do not introduce absolute paths that include a username (`/Users/<name>/...`, `/home/<name>/...`). Use `~/...` or relative paths.

### Shell (`tests/smoke.sh` etc.)

- POSIX-friendly bash with `set -euo pipefail`.
- Each check should print `PASS` / `FAIL` / `WARN` and increment a counter; final summary at the bottom.
- Optional/integration steps must be opt-in via an env var (e.g. `CODEX_HARNESS_SMOKE_INTEGRATION=1`).

## Pull request checklist

- [ ] `./tests/smoke.sh` passes locally.
- [ ] Changed manifests still parse with `jq`.
- [ ] No personal absolute paths added.
- [ ] If a new MCP tool was added, `AGENTS.md` MCP-tool table and `skills/harness/SKILL.md` are updated to match.
- [ ] If a Codex CLI behavior was newly observed, `LIMITATIONS.md` reflects the new state.
- [ ] Commit message is descriptive (English or Korean both fine).
