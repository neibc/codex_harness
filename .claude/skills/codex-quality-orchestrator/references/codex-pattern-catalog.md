# Codex 특성 패턴 카탈로그

격차 차원 → Codex-특성 가설 매핑. 사이클 돌면서 검증된 가설을 추가/수정한다.

## 1. 모델 보수성 패턴

**관측**: GPT-5.5는 같은 가이드를 받아도 phase를 적게 펼치고, 각 phase의 본문을 짧게 쓴다.

**기인 추정**:
- 모델 자체의 응답 길이 보수성
- 도구 호출 비용에 대한 자율 절약
- "최소한의 답변" 우선 학습 신호

**비침습 정비 표면 #2 — Codex 환경 박스 추가:**
```markdown
> **Codex 환경 안내**: 본 phase의 산출물은 ≥{N}줄, ≥{M}섹션을 권장합니다.
> Codex(GPT-5.x)는 명시 기준 없으면 보수적으로 끝내는 경향이 있어, 도메인이
> 깊이를 요구하면 사용자가 "최종 보고서 ≥10 섹션" 같이 정량 요구를 명시하세요.
```

**검증 가능성**: regression-tester가 같은 task에 대해 분량 측정 → 정비 후 ≥1.5× 증가 시 유효.

## 2. 변증법/대안 검토 부재 패턴

**관측**: Codex는 "주어진 문제의 답"에 수렴. devil's advocate, methodology critique 같은 자율 메타 검토 phase가 거의 없음.

**기인 추정**:
- 모델이 사용자 요청을 너무 곧이곧대로 해석
- "균형 잡힌 양측 검토" 같은 전략적 사고를 자율 추가하지 않음

**비침습 정비 표면 #2 — Codex 환경 박스:**
```markdown
> **Codex 환경 안내**: 도메인이 양측 검토(예: 옹호 vs 회의, 메타 분석)에
> 적합하면 사용자가 명시 요청해야 합니다. 예: "변증법 검토 phase 추가",
> "방법론 비평 단계 포함", "스틸맨 분석 1개 섹션"
```

## 3. 외부 자료 수집 한계 패턴

**관측**: WebFetch/WebSearch 빌트인 부재 → 자료 수집 약함 → 후속 phase 인용 빈약.

**기인**: 환경적 한계 (확정).

**비침습 정비 표면 #1 + #3 — frontmatter 강화 + 호출 wrapper:**
- 자료 수집 에이전트의 `tools:` 에 `web_search, web_fetch` 명시
- 호출 wrapper에 `--search` 플래그 또는 외부 MCP 서버 등록 안내

## 4. 도구 호출 자율 절약 패턴

**관측**: Codex는 한 번 grep/read하고 끝내는 경향. Claude는 결과를 다음 호출 input으로 연쇄.

**기인 추정**:
- `codex exec` 비용 인식
- 단발 응답 학습 편향

**비침습 정비 표면 #2 — 박스:**
```markdown
> **Codex 환경 안내**: 깊이 있는 분석에는 도구 호출 ≥{N}회 권장.
> 1차 결과만 보고 결론짓지 말고, 결과를 input으로 다음 호출을 연쇄하세요.
```

## 5. "헤더 충실, 살 부족" 패턴

**관측**: Phase 헤더는 정확히 만들지만 각 phase 본문이 1~3줄로 짧음.

**기인 추정**: 구조 따라가는 데는 강하지만 자율적으로 본문 확장은 약함.

**비침습 정비 표면 #2:**
```markdown
> **Codex 환경 안내**: 각 phase 산출물은 헤더만이 아니라 본문 ≥{N}줄,
> sub-bullet ≥{M}개를 채우세요. 본문이 짧으면 references/ 도 읽어 깊이를
> 확보하세요.
```

## 6. 종료 안내 누락 패턴

**관측**: 하네스 구성 후 "이걸 어떻게 다시 트리거하지?" 안내 출력 안 함.

**기인**: revfactory 원본의 종료 안내 단계가 본문에 추상화로만 있어 모델이 자율 판단으로 빠뜨림.

**비침습 정비 표면 #2 — 이미 적용됨**: SKILL.md에 "사용자에게 트리거링 안내 — 필수" 섹션 추가됨 (직전 commit). 효과 측정 필요.
